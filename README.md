# Clinic SMM Manager v14

Версия v9 добавляет нормальную публикацию с заголовками:

- у каждого поста теперь есть короткий `headline`;
- если пост публикуется без изображения, заголовок ставится в начало текста жирным шрифтом;
- если пост публикуется с изображением, заголовок графически встраивается в картинку;
- изображение + текст публикуются одним сообщением через Telegram caption;
- если текст слишком длинный для одного сообщения с фото, бот попросит сократить пост командой `/rewrite`.

## Важное ограничение Telegram

У подписи к фото в Telegram есть лимит около 1024 символов. Поэтому формат “изображение + полный текст одним сообщением” возможен только для постов примерно до 900–1000 символов.

Если бот выдаст ошибку лимита, сократите текст:

```text
/rewrite 4 Сократи текст до 900 символов, сохрани смысл, пользу и мягкий CTA.
```

## v10: логика публикации картинки и длинного текста

Если у поста есть изображение:

1. Заголовок встраивается в изображение.
2. Если текст помещается в лимит Telegram caption — бот публикует фото и текст одним сообщением.
3. Если текст длиннее лимита caption — бот публикует два сообщения:
   - сначала изображение с заголовком внутри;
   - затем полный пост с жирным заголовком.

Если изображения нет, бот публикует одно сообщение: жирный заголовок + текст поста.

## v13: ИИ-дизайн заголовка внутри изображения

Начиная с v13, заголовок больше не дорисовывается сервером Railway через Pillow.

Теперь при генерации картинки OpenAI сразу получает заголовок поста и создает готовую SMM-карточку:

- фотореалистичная медицинская сцена;
- дизайнерский блок заголовка внутри изображения;
- крупный читаемый русский текст;
- без отдельного серверного наложения текста поверх картинки.

Логика публикации:

1. Пост без картинки:
   - одно сообщение: жирный заголовок + текст.

2. Короткий пост с картинкой:
   - одно сообщение: картинка с заголовком внутри изображения + Telegram caption с текстом.

3. Длинный пост с картинкой:
   - первое сообщение: только картинка без Telegram caption;
   - второе сообщение: жирный заголовок + полный текст.

Если у старого поста уже была картинка, созданная до v13, для нового дизайна нажмите кнопку «Картинка» или выполните `/image ID`, чтобы перегенерировать изображение.

## v17 — анализ успешного контента

Добавлены два варианта анализа вдохновляющего контента:

### 1. Через пересылку в Telegram-бота

Команда:

```text
/inspire
```

После команды отправьте следующим сообщением:

- пересланный Telegram-пост;
- фото или картинку с подписью;
- текст успешного поста;
- сообщение со ссылкой.

Бот создаст не копию поста, а карточку вдохновения:

- идея;
- формат;
- хук;
- визуальный стиль;
- почему может сработать;
- услуга клиники;
- риски;
- новая тема для нашей клиники.

### 2. Через ссылку

Команда:

```text
/analyze_url https://example.com/post
```

Программа попробует получить title, description, текст страницы и preview-метаданные, затем создаст карточку вдохновения.

Важно: Telegram/Instagram/VK могут ограничивать доступ по ссылке. Поэтому для некоторых ссылок данных может быть мало. Самый надежный вариант на текущем этапе — переслать материал в бота через `/inspire`.

### Просмотр карточек

```text
/inspirations
```

### План на основе карточек вдохновения

```text
/plan_week_from_inspirations
```

После этого можно создавать посты по пунктам плана:

```text
/create_full_from_plan ID
```
