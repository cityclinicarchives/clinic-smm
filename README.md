# clinic-smm-manager v25

Версия v25 добавляет **Blueprint Layout Validator + Multi-format Layout Planner** поверх crop-and-assemble архитектуры v24.

## Главное изменение

В v24 программа уже пыталась физически вырезать элементы из исходника и собрать новую инфографику, но layout оставался слишком жестким: квадрат, 3 колонки и фиксированная высота карточек. Из-за этого часть блоков не помещалась.

В v25 pipeline такой:

```text
/analyze_asset
→ исходник сохраняется
→ /reconstruct_asset ID
→ ИИ создает structured blueprint
→ blueprint содержит canvas + blocks + layout + source_bbox
→ validator проверяет layout
→ если layout плохой, включается auto multi-format planner
→ /create_full_from_reconstruction ID
→ программа физически вырезает source_bbox из исходника
→ генерирует только недостающие/замененные элементы
→ собирает новую инфографику в подходящем формате
```

## Multi-format planner

Программа выбирает формат по сложности материала:

```text
1:1  — простая карточка / Instagram square
4:5  — основной Instagram feed
3:4  — Telegram/Pinterest инфографика
2:3  — высокая сложная инфографика
9:16 — stories/reels cover
```

Если блоков много, программа больше не пытается запихнуть всё в квадрат.

## Новые поля blueprint

Для каждого блока ИИ должен указывать не только `source_bbox`, но и `layout`:

```json
{
  "id": "mosquito",
  "type": "comparison_card",
  "title": "Комар",
  "lines": ["Зудящий волдырь", "Часто слабый зуд"],
  "visual_element": "комар + типичный след укуса",
  "source_policy": "preserve_from_reference",
  "source_bbox": {"x": 0.05, "y": 0.18, "w": 0.28, "h": 0.20},
  "layout": {"x": 0.03, "y": 0.14, "w": 0.30, "h": 0.20},
  "replacement_prompt": "",
  "change_reason": "хороший визуальный пример в исходнике"
}
```

`source_bbox` — откуда брать элемент на исходной картинке.  
`layout` — куда поставить блок на новой инфографике.

## Validator

Перед рендером программа проверяет:

```text
- есть ли карточки;
- совпадает ли expected_block_count;
- есть ли source_bbox для preserve/use_reference;
- есть ли layout;
- помещаются ли блоки в canvas;
- не пересекаются ли блоки.
```

Если AI layout плохой, программа не падает, а строит безопасный layout сама.

## Команды

```text
/analyze_asset
```
Загрузить исходник: инфографику, пост, мем, скриншот.

```text

```

```text

```

## v26 — Strict Blueprint Validator + Multi-format Escalation

- prompt инфографики теперь требует полный список смысловых блоков, `expected_block_count`, `source_bbox`, `old_element/new_element` для замен.

## v27 — Atomic Blueprint + Crop Sanity Check + Blueprint Repair Loop

В этой версии исправлен главный недостаток v26: программа больше не должна принимать один большой блок «вся сетка/таблица/коллаж» вместо отдельных смысловых элементов.

Добавлено:

- `app/services/atomic_blueprint.py`
- проверка, что каждый visual/comparison блок атомарный;
- запрет слишком больших `source_bbox` для карточек;
- repair-loop: если AI вернул неатомарный blueprint, программа отправляет JSON и исходную картинку на исправление до рендера;
- поддержка `atomic_blueprint.content_units`;
- дополнительные правила в промпте инфографики;
- поиск DejaVu-шрифтов из matplotlib, чтобы уменьшить риск квадратов вместо кириллицы в Pillow-render.

## v28: Crop Preview перед финальной сборкой инфографики

Для реконструкций с режимом `crop_and_assemble_infographic` команда/кнопка **Пост + картинка** теперь сначала создает debug-preview:

1. показывает, какие атомарные блоки программа реально вырезала из исходника;
2. показывает, какие блоки будут сгенерированы заново;
3. выводит критические проблемы blueprint/crop;
4. финальная сборка запускается только после кнопки **✅ Собрать инфографику**.

Если preview показывает неверные crop-блоки, лучше нажать **🧠 Реконструировать заново** или заново выполнить `/reconstruct_asset ID` с уточнением.

## v29 — Technical Draft + AI Design Polish

В этой версии для сложных инфографик добавлен новый двухэтапный pipeline:

1. **Technical Draft Renderer** — программа сначала собирает точный черновик из atomic-блоков: crop-элементы, заголовки, warning/action/prevention-блоки. Главная цель этого этапа — не красота, а полнота: все блоки должны быть на месте и не обрезаны.
2. **AI Design Polish** — затем ИИ получает технический черновик как референс и улучшает только дизайн: композицию, отступы, карточки, цвета и общий стиль. ИИ запрещено удалять блоки, менять смысл и ломать русские подписи.
3. **Fallback** — если дизайн-полировка не сработает, бот вернет технический черновик, чтобы не потерять структуру и содержание.
4. **Reconstruction Post Writer** — пост теперь пишется не по общей теме, а по полному structured reconstruction spec: что показывает инфографика, ограничения, когда обращаться к врачу, безопасные действия и профилактика.

## v30 — Technical Draft + AI Polish + Final QA + выбор версии

В этой версии после дизайн-полировки добавлен обязательный QA-этап:

1. программа собирает точный технический черновик;
2. ИИ делает дизайн-версию;
3. отдельная QA-проверка сравнивает черновик и дизайн-версию;
4. если дизайн-версия потеряла блоки, обрезала текст или сломала кириллицу — автоматически выбирается технический черновик;
5. бот отправляет обе версии и дает кнопки:
   - **Взять дизайн-версию**;
   - **Взять точный черновик**.

Это защищает от ситуации, когда красивая версия выглядит лучше, но теряет медицинскую точность или структуру.

## v31 — Reconstruction Contract

Добавлен строгий semantic contract для реконструкций:

- `required_elements` — элементы, которые обязаны попасть в итоговую инфографику;
- `forbidden_elements` — элементы, которые нельзя оставлять в финальной версии;
- `replacement_rules` — обязательные замены, например `Скорпион -> Слепень/мошка`;
- `expected_atomic_cards` — ожидаемое число атомарных карточек;
- `required_blocks` — обязательные служебные блоки: предупреждения, действия, профилактика.

Если contract не выполнен, программа останавливает сборку и пишет ошибку вместо публикации неправильной инфографики.

## v32 update

- Запрещен небезопасный fallback для сложных crop-and-assemble инфографик: если строгая сборка ломается, генерация останавливается с ошибкой, а не уходит в обычную картинку.
- Crop-preview для сложных инфографик обязателен перед финальной сборкой.
- Final QA усилен проверкой required_elements, forbidden_elements и replacement_rules по фактической design-версии.
- Добавлен обязательный post_brief в промпт реконструкции: пост пишется не как общая статья, а как сопровождение к конкретной финальной инфографике.
- Бот по-прежнему показывает технический черновик и design-версию, чтобы можно было вручную выбрать лучший вариант.

## v33 update

- Усилены правила crop: для карточек сохраняется только полезный визуал, без подписей, синего фона, watermark и UI соцсетей.

- Промпт инфографики теперь требует `crop_target`, `exclude_from_crop`, `visual_cleanup`, `remove_source_labels` и более дизайнерский layout plan.

## v34 — Semantic Crop + Text Overflow Protection

В этой версии усилен pipeline реконструкции инфографик:

1. **AI Crop Intent** — blueprint теперь должен описывать не только `source_bbox`, но и `crop_intent`: что оставить, что удалить, какой объект главный.
2. **Semantic Crop Cleaner** — перед сборкой Python пытается вырезать из crop только полезный визуал: например, круг укуса + насекомое, без старых подписей, синего фона, watermark и интерфейса соцсетей.
3. **Crop Quality Check** — программа помечает подозрительные crop как критические: слишком узкие, слишком маленькие, похожие на screenshot-strip.
4. **No Cover-Crop in Draft** — подготовленный визуальный компонент больше не обрезается повторно при размещении в карточке.
5. **Text Overflow Protection** — увеличены минимальные размеры карточек и footer-блоков, добавлены ограничения на вывод текста, чтобы черновик не обрезал строки.
6. **Prompt Rules** — промпт теперь заставляет ИИ давать короткие строки для инфографики и переносить подробные объяснения в текстовый пост.


## v35

Добавлен AI Crop Planner: source_bbox больше не используется напрямую. Программа сначала строит отдельный crop_plan, поддерживает multi_part_bbox, очищает visual-компоненты через AI, показывает clean crop-preview и не использует dirty raw crop в финальной сборке.
