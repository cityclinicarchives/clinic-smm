# clinic-smm-manager v21

AI-SMM backend для медицинской клиники: генерация постов, изображений, согласование, публикация в тестовую Telegram-группу, анализ контента, паттерны и structured reconstruction.

## Что нового в v21

### 1. Structured Reconstruction Engine

Реконструкция теперь сохраняется не только как текстовый анализ, а как структурированный JSON blueprint:

- оценка исходного заголовка;
- что сохранить;
- что исправить;
- медицинский аудит;
- добавления по теме;
- финальная структура инфографики/поста;
- визуальная стратегия;
- готовый текст поста;
- режим визуала: `deterministic_infographic` или `ai_image`.

Это нужно, чтобы пост и картинка создавались из одного и того же плана и не теряли смысл реконструкции.

### 2. Hybrid Infographic Engine

Для инфографик бот больше не просит image AI писать русский текст внутри картинки.

Теперь логика такая:

```text
AI → structured blueprint → Python/Pillow renderer → итоговая инфографика
```

Это уменьшает ошибки в русских словах, подписях и структуре.

### 3. Исправлены кнопки

Теперь работают кнопки:

- 🧠 Реконструировать;
- 📝 Создать пост;
- 🖼 Пост + картинка.

## Основной workflow

```text
/analyze_asset
```

Отправьте инфографику, пост, мем, скриншот или текст.

```text
/reconstruct_asset ID
```

Создает structured reconstruction blueprint.

```text
/create_from_reconstruction ID
```

Создает текстовый пост.

```text
/create_full_from_reconstruction ID
```

Создает пост + визуал. Если это инфографика, визуал собирается deterministic renderer-ом.

```text
/approve ID_поста
/publish ID_поста
```

Одобрение и публикация в тестовую группу.
