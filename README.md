# Clinic SMM Manager v18

Версия v18 добавляет новый аналитический слой поверх v17:

```text
исходный контент → паттерн внимания → контекст → новая оригинальная идея для клиники
```

Теперь система сохраняет не только inspiration-карточки, но и сам исходный материал, механику вовлечения и контекст, из-за которого материал может работать.

## Новые таблицы

```text
content_assets
content_patterns
content_contexts
```

## Новые команды Telegram

```text
/analyze_asset
```

Включает режим анализа материала. После команды отправьте боту:

- мем или скриншот;
- пересланный Telegram-пост;
- фото/картинку с подписью;
- текст поста;
- ссылку с пояснением.

Бот сохранит 3 слоя:

```text
1. исходный контент;
2. паттерн внимания;
3. контекст и применимость для клиники.
```

```text
/assets
```

Показать последние сохраненные исходники.

```text
/patterns
```

Показать последние найденные паттерны.

```text
/generate_from_pattern ID
```

Создать новый медицинский пост на основе выбранного паттерна, без копирования исходного материала.

## v19: Creative Visual Engine

В этой версии генерация изображений больше не строится по однотипному шаблону `врач + пациент + подпись`.
Перед генерацией картинки система создает отдельную визуальную концепцию:

- эмоция изображения;
- visual hook;
- уместность юмора;
- визуальная метафора;
- формат: фото / инфографика / постер / комикс / чек-лист / символическая сцена;
- композиция;
- ключевые объекты;
- ограничения медицинской безопасности.

Основной новый файл:

```text
app/services/visual_concept_engine.py
```

Обновленный файл:

```text
app/services/image_generator.py
```

Команды остаются прежними:

```text
/generate_full тема
/image ID
```

Если картинка получилась слишком обычной, можно перегенерировать ее с уточнением:

```text
/image 12 сделай с юмором: спортсмен смотрит на огромный бургер как на спортивный снаряд
```

или:

```text
/image 12 сделай как чек-лист с пробирками, часами, стаканом воды и перечеркнутым кофе
```

## v20 — Universal Content Reconstruction Engine

В этой версии добавлен новый слой анализа: **экспертная реконструкция контента**.

Главная идея: бот больше не просто извлекает паттерн и генерирует новый пост. Он сохраняет и анализирует связку:

```text
исходный контент → хук → эмоция → медицинская корректность → недостающая польза → новая улучшенная версия
```

### Новые команды Telegram

```text
/reconstruct_asset ID
```

Экспертно реконструирует сохраненный контент-исходник. Например:

```text
/reconstruct_asset 1
```

Бот анализирует:

- стоит ли сохранять исходный заголовок;
- что в исходнике медицински корректно;
- что нужно исправить;
- какие полезные дополнения можно добавить;
- как сохранить работающий паттерн, но сделать оригинальный материал для клиники;
- какой визуал лучше создать.

```text
/reconstructions
```

Показывает последние реконструкции.

```text
/create_from_reconstruction ID
```

Создает текстовый пост по реконструкции.

```text
/create_full_from_reconstruction ID
```

Создает пост и изображение по реконструкции.

### Рекомендуемый workflow

```text
/analyze_asset
```

Отправьте инфографику, мем, скриншот, пост или ссылку.

Затем:

```text
/reconstruct_asset ID
```

Затем:

```text
/create_full_from_reconstruction ID
```

После этого используйте обычные кнопки под постом: редактировать, одобрить, опубликовать.
