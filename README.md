# clinic-smm-manager v24

Версия v24 добавляет настоящий `crop-and-assemble` алгоритм для сложных инфографик.

## Главное изменение

Раньше программа просила ИИ «представить» компонентную архитектуру и затем генерировала всю инфографику целиком. Теперь для инфографик pipeline такой:

```text
/analyze_asset
→ исходная инфографика сохраняется
→ /reconstruct_asset ID
→ ИИ создает structured blueprint
→ для каждого блока указывается source_policy и source_bbox
→ /create_full_from_reconstruction ID
→ программа физически вырезает нужные элементы из исходника
→ генерирует новые элементы там, где нужно заменить
→ собирает новую инфографику локально через Pillow
```

## Важные поля blueprint

Для каждого блока инфографики ИИ теперь должен указать:

```json
{
  "id": "mosquito",
  "type": "comparison_card",
  "title": "Комар",
  "lines": ["Зудящий волдырь", "Часто слабый зуд"],
  "visual_element": "комар + типичный след укуса",
  "source_policy": "preserve_from_reference",
  "source_bbox": {"x": 0.05, "y": 0.18, "w": 0.28, "h": 0.20},
  "replacement_prompt": "",
  "change_reason": "хороший визуальный пример в исходнике"
}
```

`source_bbox` — нормализованные координаты на исходной картинке от 0 до 1.

## Source policy

```text
preserve_from_reference — вырезать из исходника и сохранить
use_reference_and_clean — вырезать из исходника, очистить/встроить в новый дизайн
replace_with_new — не использовать исходник, сгенерировать новый визуал
 недостающий/новый блок — generate_new
text_only — только текстовый блок
```

## Команды

```text
/analyze_asset
```
Загрузить исходник: инфографику, пост, мем, скриншот.

```text
/reconstruct_asset ID
```
Создать structured reconstruction blueprint.

```text
/create_full_from_reconstruction ID
```
Создать пост + новую инфографику по blueprint.

## Что v24 пока не делает идеально

- Автоматическое определение координат `source_bbox` зависит от качества vision-анализа.
- Если ИИ не дал bbox, программа пытается угадать позицию по `row/col`, но это fallback.
- Для новых замененных элементов программа может делать отдельные image API вызовы, поэтому сложные инфографики могут стоить дороже.
